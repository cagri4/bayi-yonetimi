**Step 1: Register the Custom Access Token Hook**

1. Navigate to: https://supabase.com/dashboard/project/neqcuhejmornybmbclwt/auth/hooks
2. Find "Custom Access Token" (or "Customize Access Token") hook
3. Click "Add hook" or the toggle to enable it
4. Set Hook Type: "Postgres function"
5. Set Schema: "public"
6. Set Function: "inject_company_claim"
7. Click "Save"

The hook is now active — every new JWT (on login or token refresh) will call this function.

**Step 2: Verify JWT injection for dealer user**

Option A — Via Supabase SQL Editor (easier):
```sql
-- Run this in SQL Editor while NOT impersonating any user
-- It won't show current_company_id() without a real JWT
-- Instead, verify the function exists and has correct grants:
SELECT routine_name, routine_type
FROM information_schema.routines
WHERE routine_name = 'inject_company_claim';
-- Should return 1 row
```

Option B — Via the application (definitive test):
1. Open https://bayi-yonetimi.vercel.app in a private/incognito browser window
2. Log in as bayi@test.com (dealer user)
3. Open browser DevTools > Application > Local Storage > find supabase auth token
4. Copy the `access_token` value
5. Decode it at https://jwt.io
6. In the "Payload" section, confirm these fields are present:
   - `"company_id"`: a UUID string (not null, not empty)
   - `"user_role"`: `"dealer"`

**Step 3: Verify JWT for admin user**

1. Log in as admin@test.com in a private browser window
2. Decode the JWT at https://jwt.io
3. Confirm:
   - `"company_id"`: a UUID string (should match the Default Company's UUID)
   - `"user_role"`: `"admin"`

**Step 4: Verify company isolation via API**

Still logged in as admin@test.com, open browser DevTools > Console and run:
```javascript
// This tests that the admin can only see their own company's dealers
const { data, error } = await window.__supabase.from('dealers').select('id, company_name, company_id').limit(5)
console.log(data, error)
// All returned rows should have the same company_id as the admin's JWT company_id
// If error appears (empty results), the RLS policies are working but the hook might not be injecting correctly
```

If the application doesn't expose __supabase globally, alternatively:
1. In Supabase Dashboard > Table Editor, switch to the 'dealers' table
2. Confirm rows are visible with company_id populated

**Step 5: Run cross-company isolation verification SQL**

In Supabase Dashboard SQL Editor, run:
```sql
-- Verify all tenant tables have company_id NOT NULL
SELECT
  table_name,
  COUNT(*) FILTER (WHERE company_id IS NULL) AS null_count,
  COUNT(*) AS total_rows
FROM (
  SELECT 'dealers' AS table_name, company_id FROM dealers
  UNION ALL SELECT 'orders', company_id FROM orders
  UNION ALL SELECT 'products', company_id FROM products
  UNION ALL SELECT 'dealer_transactions', company_id FROM dealer_transactions
  UNION ALL SELECT 'announcements', company_id FROM announcements
  UNION ALL SELECT 'support_messages', company_id FROM support_messages
) combined
GROUP BY table_name
ORDER BY table_name;
-- Expected: null_count = 0 for all tables
```
