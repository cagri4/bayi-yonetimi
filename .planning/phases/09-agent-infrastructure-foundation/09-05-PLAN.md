**Step 2: Create webhook route (src/app/api/telegram/route.ts)**

This is a THIN adapter. It does three things: parse, dedup, dispatch. No agent logic in this file.

```typescript
import { after } from 'next/server'
import type { Update } from 'grammy/types'
import { createServiceClient } from '@/lib/supabase/service-client'
import { dispatchAgentUpdate } from '@/lib/agents/dispatcher'

export const dynamic = 'force-dynamic'
```

**Export: `async function POST(request: Request): Promise<Response>`**

1. Parse the request body as JSON. Type it as `Update` (from grammy/types). If parsing fails, return `new Response('Bad Request', { status: 400 })`.

2. Idempotency check (AI-05):
   - `const supabase = createServiceClient()`
   - Insert into processed_telegram_updates: `{ update_id: update.update_id }`
   - If error code is '23505' (unique_violation): return `new Response('OK', { status: 200 })` — this is a duplicate
   - If any other error: log it with `console.error('[telegram] idempotency insert error:', error)`, still return 200 (prevent Telegram retry storm)

3. Background dispatch (AI-04):
   - `after(async () => { try { await dispatchAgentUpdate(update) } catch (err) { console.error('[telegram] dispatch error:', err) } })`

4. Return 200 immediately:
   - `return new Response('OK', { status: 200 })`

**Critical:** The idempotency INSERT happens BEFORE the return statement (synchronous part). The dispatchAgentUpdate runs AFTER the response is sent via after().

**Do NOT import Bot from grammy** — we parse the update manually via `request.json()`. No Bot instance needed in the webhook route.

Create the dispatcher at `src/lib/agents/dispatcher.ts`. This is the orchestrator that wires all agent components together.

**Imports:**
- `Update` from `grammy/types`
- `createServiceClient` from `@/lib/supabase/service-client`
- `AgentRunner` from `./agent-runner`
- `ConversationManager` from `./conversation-manager`
- `TokenBudget` from `./token-budget`
- `ToolRegistry` from `./tool-registry`
- `AgentContext, AgentRole` from `./types`

**Export: `async function dispatchAgentUpdate(update: Update): Promise<void>`**

This function runs inside `after()` — it has up to 300 seconds on Vercel Fluid Compute.

**Step 1: Extract message from update**
- Get the text message: `const message = update.message`
- If no message or no message.text, return early (ignore non-text updates like edits, callbacks, etc.)
- Extract: `chatId = message.chat.id`, `text = message.text`, `telegramUserId = message.from?.id`

**Step 2: Resolve dealer identity**
- Query dealers table via service client: find dealer where telegram_chat_id matches chatId
- If no dealer found: send a reply via Telegram API saying "Bu bot'u kullanmak icin hesabinizi baglayiniz. Bayi panelinizden Telegram entegrasyonunu tamamlayin." and return
- Extract: dealerId, companyId from the dealer row

**Step 3: Determine agent role**
- For Phase 9, the webhook handles a single bot. The agent role is determined by which bot received the message.
- For now, use a default: check if the update has `message.chat` — look up agent_definitions by company_id where is_active = true to find the role mapped to this bot. If none found, use 'destek' as fallback.
- In Phase 10+, each bot will have its own webhook URL or a bot-to-role mapping table.

**Step 4: Check token budget**
- `const tokenBudget = new TokenBudget()`
- `const budgetCheck = await tokenBudget.checkBudget(dealerId)`
- If not allowed: send Telegram reply with budgetCheck.reason, return

**Step 5: Load conversation and messages**
- `const conversationManager = new ConversationManager()`
- `const conversationId = await conversationManager.getOrCreateConversation({ companyId, dealerId, agentRole: role, telegramChatId: chatId })`
- Save the incoming user message: `await conversationManager.saveMessage(conversationId, 'user', text)`
- Load message history: `const messages = await conversationManager.getMessages(conversationId)`

**Step 6: Build and run agent**
- `const toolRegistry = new ToolRegistry()`
- `const tools = toolRegistry.getTools(role as AgentRole)`
- `const model = toolRegistry.getModel(role as AgentRole)`
- Create tool handlers map — for Phase 9, map the placeholder tool names to simple handler functions:
  - 'echo': returns the input message
  - 'get_current_time': returns new Date().toISOString()
  - 'lookup_dealer': queries dealer info via service client (scoped by companyId)
- Build AgentContext: `{ companyId, dealerId, conversationId, agentRole: role, telegramChatId: chatId, callStack: [role], depth: 0 }`
- Get system prompt from agent_definitions table (or use a default: "Sen bir bayi yardim asistanisin. Turkce cevap ver. Bayi sorularina yardimci ol.")
- `const runner = new AgentRunner(model, tools, toolHandlers)`
- `const reply = await runner.run(systemPrompt, messages, context)`

**Step 7: Save and send reply**
- Save assistant reply: `await conversationManager.saveMessage(conversationId, 'assistant', reply)`
- Send Telegram reply via direct API call:
  ```typescript
  await fetch(`https://api.telegram.org/bot${process.env.TELEGRAM_BOT_TOKEN}/sendMessage`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ chat_id: chatId, text: reply, parse_mode: 'Markdown' }),
  })
  ```

**Error handling:**
- Wrap the entire function in try/catch
- On error: log with `console.error('[dispatcher] error:', err)`
- Attempt to send an error message to the user via Telegram: "Bir hata olustu. Lutfen tekrar deneyin."
- Wrap the error message send in its own try/catch (don't fail on the error notification)

**Helper: `async function sendTelegramMessage(chatId: number, text: string): Promise<void>`**
- Extract the Telegram API call into a reusable helper within the same file
- Use `process.env.TELEGRAM_BOT_TOKEN`
- Handle API errors gracefully (log but don't throw)
