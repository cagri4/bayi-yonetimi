  <files>supabase/migrations/010_agent_tables.sql</files>
  <action>
Create migration file `supabase/migrations/010_agent_tables.sql` with the following tables and objects. Follow existing migration conventions (gen_random_uuid(), TIMESTAMPTZ DEFAULT NOW(), company_id FK to companies with ON DELETE CASCADE).

**Tables to create (6 total):**

1. **agent_definitions** — Role catalog (one row per company+role pair)
   - id UUID PK DEFAULT gen_random_uuid()
   - company_id UUID NOT NULL REFERENCES companies(id) ON DELETE CASCADE
   - role TEXT NOT NULL
   - name TEXT NOT NULL
   - description TEXT
   - model TEXT NOT NULL DEFAULT 'claude-haiku-4-5'
   - system_prompt TEXT NOT NULL DEFAULT ''
   - is_active BOOLEAN DEFAULT true
   - settings JSONB DEFAULT '{}'
   - created_at TIMESTAMPTZ DEFAULT NOW()
   - updated_at TIMESTAMPTZ DEFAULT NOW()
   - UNIQUE (company_id, role)

2. **agent_conversations** — One per dealer-bot thread
   - id UUID PK DEFAULT gen_random_uuid()
   - company_id UUID NOT NULL REFERENCES companies(id) ON DELETE CASCADE
   - dealer_id UUID NOT NULL REFERENCES dealers(id) ON DELETE CASCADE
   - agent_role TEXT NOT NULL
   - telegram_chat_id BIGINT
   - status TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'closed', 'error'))
   - summary TEXT
   - metadata JSONB DEFAULT '{}'
   - created_at TIMESTAMPTZ DEFAULT NOW()
   - updated_at TIMESTAMPTZ DEFAULT NOW()

3. **agent_messages** — Individual messages in a conversation
   - id UUID PK DEFAULT gen_random_uuid()
   - conversation_id UUID NOT NULL REFERENCES agent_conversations(id) ON DELETE CASCADE
   - role TEXT NOT NULL CHECK (role IN ('user', 'assistant', 'system'))
   - content TEXT NOT NULL
   - metadata JSONB DEFAULT '{}' (for token counts, tool_use IDs)
   - created_at TIMESTAMPTZ DEFAULT NOW()

4. **agent_calls** — Cross-agent call audit log
   - id UUID PK DEFAULT gen_random_uuid()
   - company_id UUID NOT NULL REFERENCES companies(id) ON DELETE CASCADE
   - conversation_id UUID REFERENCES agent_conversations(id) ON DELETE SET NULL
   - caller_role TEXT NOT NULL
   - callee_role TEXT NOT NULL
   - depth INTEGER NOT NULL DEFAULT 0
   - success BOOLEAN NOT NULL DEFAULT true
   - error_message TEXT
   - created_at TIMESTAMPTZ DEFAULT NOW()

5. **processed_telegram_updates** — Idempotency log
   - update_id BIGINT PRIMARY KEY (NOT auto-generated; Telegram provides this)
   - processed_at TIMESTAMPTZ DEFAULT NOW()

6. **daily_token_usage** — Per-dealer daily budget tracking
   - dealer_id UUID NOT NULL REFERENCES dealers(id) ON DELETE CASCADE
   - date DATE NOT NULL
   - tokens_used INTEGER NOT NULL DEFAULT 0
   - updated_at TIMESTAMPTZ DEFAULT NOW()
   - PRIMARY KEY (dealer_id, date)

**RLS policies:**
- agent_definitions: Company admins can manage (FOR ALL, company_id = current_company_id() AND is_company_admin()). Service role full access. Superadmin full access.
- agent_conversations: Service role full access only (agents run via service role, not user sessions).
- agent_messages: Service role full access only.
- agent_calls: Service role full access only. Company admins can SELECT (for audit viewing).
- processed_telegram_updates: Service role full access only (no RLS needed — no company_id column).
- daily_token_usage: Service role full access only.

**Indexes:**
- idx_agent_conversations_dealer ON agent_conversations(company_id, dealer_id)
- idx_agent_conversations_chat ON agent_conversations(telegram_chat_id)
- idx_agent_messages_conversation ON agent_messages(conversation_id, created_at)
- idx_agent_calls_company ON agent_calls(company_id, created_at DESC)
- idx_daily_token_usage_dealer ON daily_token_usage(dealer_id, date DESC)

**RPC function:**
- increment_daily_token_usage(p_dealer_id UUID, p_date DATE, p_tokens INTEGER) RETURNS VOID — atomic INSERT ON CONFLICT DO UPDATE SET tokens_used = daily_token_usage.tokens_used + EXCLUDED.tokens_used, updated_at = NOW(). Use LANGUAGE sql.

Label each section with clear comments (-- BLOCK 1: agent_definitions, etc.) for Dashboard SQL Editor execution.
