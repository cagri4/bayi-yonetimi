  <action>
Create the AgentRunner class at `src/lib/agents/agent-runner.ts`. This class drives the Claude tool-use loop — the core execution engine for all agents.

**Imports:**
- `Anthropic` default import from `@anthropic-ai/sdk`
- Types: `MessageParam, Tool, ContentBlock` from `@anthropic-ai/sdk/resources/messages` (or from the SDK's type exports — check what the SDK actually exports)
- `AgentContext, TokenUsageRecord, MAX_ITERATIONS` from `./types`
- `TokenBudget` from `./token-budget`
- `applyPromptCaching` from `./tool-registry`

**Constructor parameters:**
- `model: string` — Claude model ID
- `tools: Tool[]` — Tool definitions for this agent role
- `toolHandlers: Map<string, (input: Record<string, unknown>, context: AgentContext) => Promise<string>>` — Maps tool name to handler function. Each handler receives the parsed input AND the AgentContext (for companyId scoping).

**Private members:**
- `client: Anthropic` — instantiated with `process.env.ANTHROPIC_API_KEY!`
- `tokenBudget: TokenBudget` — instantiated once

**Method: `async run(systemPrompt: string, messages: MessageParam[], context: AgentContext): Promise<string>`**

Implementation:
1. Clone the messages array: `const workingMessages = [...messages]`
2. Apply prompt caching to tools: `const cachedTools = applyPromptCaching(this.tools)`
3. Enter the tool-use loop: `while (iterations < MAX_ITERATIONS)`
4. On each iteration:
   a. Call `this.client.messages.create()` with:
      - model: this.model
      - max_tokens: 4096
      - system: `[{ type: 'text', text: systemPrompt, cache_control: { type: 'ephemeral' } }]` (AI-10 prompt caching on system)
      - tools: cachedTools
      - messages: workingMessages
   b. Record token usage via `this.tokenBudget.recordUsage(context.dealerId, { inputTokens, outputTokens, cacheReadTokens: cache_read_input_tokens ?? 0, cacheWriteTokens: cache_creation_input_tokens ?? 0 })`
   c. If `response.stop_reason === 'end_turn'`: extract text from content blocks, return it
   d. If `response.stop_reason === 'tool_use'`:
      - Push assistant content to workingMessages
      - Iterate over content blocks, find type === 'tool_use' blocks
      - For each tool_use block: look up handler in toolHandlers map, call it with (input, context), catch errors
      - Push tool_result blocks as a user message
      - Continue loop
   e. If stop_reason is 'max_tokens' or anything else: break out of loop
5. After loop: return `'[Maksimum islem adimina ulasildi. Lutfen tekrar deneyin.]'` (Turkish per convention)

**Error handling:**
- Wrap individual tool handler calls in try/catch. On error, return the error message as tool_result content (not throw).
- Do NOT wrap the entire messages.create() call — let API errors propagate to the caller (dispatcher handles them).

**Important details:**
- The `cache_control` on system prompt uses the array format for system (array of content blocks with cache_control), not a plain string.
- Tool handlers receive AgentContext so they can enforce company_id scoping.
- Token usage is recorded AFTER every API call, even if it is followed by more iterations.

<task type="auto">
  <name>Task 2: Create ConversationManager class</name>
  <files>src/lib/agents/conversation-manager.ts</files>
  <action>
Create the ConversationManager class at `src/lib/agents/conversation-manager.ts`. This class manages DB-backed conversation history with a rolling 50-message window and automatic Haiku summarization.

**Imports:**
- `createServiceClient` from `@/lib/supabase/service-client`
- `Anthropic` from `@anthropic-ai/sdk`
- `MessageParam` from `@anthropic-ai/sdk/resources/messages`

**Constants:**
- `ROLLING_WINDOW = 50`
- `SUMMARIZE_THRESHOLD = 50`
- `KEEP_RECENT = 25` (messages to keep after summarization)

**Method 1: `async getOrCreateConversation(params: { companyId: string, dealerId: string, agentRole: string, telegramChatId: number }): Promise<string>`**
- Query agent_conversations for an active conversation matching (company_id, dealer_id, agent_role, telegram_chat_id, status='active')
- If found, return the conversation id
- If not found, insert a new row and return the new id
- This is called once per incoming Telegram message

**Method 2: `async getMessages(conversationId: string): Promise<MessageParam[]>`**
- Query agent_messages ordered by created_at ASC, limit ROLLING_WINDOW
- Map to MessageParam format: `{ role: m.role as 'user' | 'assistant', content: m.content }`
- Filter out 'system' role messages (summaries are stored as system but injected differently)
- Return the array

**Method 3: `async saveMessage(conversationId: string, role: 'user' | 'assistant', content: string, metadata?: Record<string, unknown>): Promise<void>`**
- Insert into agent_messages: conversation_id, role, content, metadata (default {})
- After insert, check total message count for this conversation
- If count > SUMMARIZE_THRESHOLD, call `this.summarizeAndTruncate(conversationId)`

**Method 4: `private async summarizeAndTruncate(conversationId: string): Promise<void>`**
- Fetch ALL messages for the conversation ordered by created_at ASC
- If count <= ROLLING_WINDOW, return early
- Split: `toSummarize = allMessages.slice(0, -KEEP_RECENT)`, `toKeep = allMessages.slice(-KEEP_RECENT)`
- Call Claude Haiku 4.5 for summarization:
  - system: 'Summarize the following conversation history concisely in 3-5 sentences. Focus on key decisions, requests, and outcomes. Respond in Turkish.'
  - user message: join toSummarize messages as "role: content" lines
  - max_tokens: 1024
- Extract summary text from response
- Delete summarized messages by their IDs
- Insert a new 'system' role message with content `[Konusma ozeti: ${summary}]` and created_at set to the earliest deleted message's timestamp (to maintain chronological order)
- Update the conversation's `summary` field with the latest summary text

**Method 5: `async getSummary(conversationId: string): Promise<string | null>`**
- Query agent_conversations for the summary field
- Return it (may be null if no summarization has occurred)

**Important details:**
- The summarization call uses Haiku 4.5 (cheapest model) regardless of the agent's assigned model.
- Turkish language for summarization prompt since all conversations are in Turkish.
- The summary message uses role='system' in the DB but is NOT passed to Claude as a system message. Instead, the dispatcher prepends it to the user's first message or the system prompt.
- Error handling: wrap summarization in try/catch — if it fails, log the error but do NOT block the main conversation flow.
