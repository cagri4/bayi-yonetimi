**Interface: AgentCallContext** (exported)
```typescript
export interface AgentCallContext {
  callStack: string[]    // Chain of roles: ['satis_temsilcisi', 'depo_sorumlusu', ...]
  depth: number          // Current recursion depth (starts at 0)
  toolCallCount: number  // Total cross-agent tool calls in this chain (for AI-08 cap)
  parentCallId?: string  // References agent_calls.id for the parent call
}
```

**Class: AgentBridge**

**Private member:** `supabase` — created via `createServiceClient()` (lazy singleton)

**Method 1: `checkDeadlock(targetRole: string, context: AgentCallContext): { allowed: boolean; reason?: string }`**
- Cycle detection: if `context.callStack.includes(targetRole)`, return `{ allowed: false, reason: \`Dongu tespit edildi: ${context.callStack.join(' -> ')} -> ${targetRole}\` }`
- Depth limit: if `context.depth >= MAX_AGENT_DEPTH`, return `{ allowed: false, reason: \`Maksimum ajan derinligi (${MAX_AGENT_DEPTH}) asildi. Cagri zinciri: ${context.callStack.join(' -> ')}\` }`
- Tool call cap: if `context.toolCallCount >= MAX_TOOL_CALLS`, return `{ allowed: false, reason: \`Maksimum cross-agent tool call sayisina (${MAX_TOOL_CALLS}) ulasildi.\` }`
- Otherwise return `{ allowed: true }`
- This method is SYNCHRONOUS (no DB call needed) — it only checks the in-memory call stack.

**Method 2: `async logAgentCall(params: { callerRole: string, calleeRole: string, depth: number, companyId: string, conversationId: string, success: boolean, errorMessage?: string }): Promise<string>`**
- Insert into agent_calls table with all provided fields
- Return the new row's id (via .select('id').single())
- If insert fails, log error and return empty string (do not throw)

**Method 3: `async callAgent(targetRole: string, query: string, context: AgentCallContext & { companyId: string, conversationId: string }): Promise<{ success: boolean; result?: string; error?: string }>`**
- This is the high-level orchestration method. In Phase 9, it only does deadlock checking and logging. The actual agent invocation (calling another AgentRunner) will be wired in Phase 10+.
- Steps:
  1. Call `this.checkDeadlock(targetRole, context)` — if not allowed, log the failed call and return `{ success: false, error: reason }`
  2. Log the call attempt via `this.logAgentCall({ callerRole: context.callStack[context.callStack.length - 1], calleeRole: targetRole, depth: context.depth, companyId: context.companyId, conversationId: context.conversationId, success: true })`
  3. For Phase 9: return `{ success: true, result: '[Cross-agent call placeholder - will be implemented in Phase 10]' }`
  4. In Phase 10+, step 3 will be replaced with actual AgentRunner invocation using the updated callStack and depth+1

**Method 4: Direct DB query helpers (for cross-agent data lookups that do NOT need Claude reasoning):**

These are simple read-only queries scoped by companyId. They demonstrate the "direct DB query, not Claude invocation" pattern from AI-06.

- Return array of order summaries (id, order_number, status, total_amount, created_at)

```typescript

```
- Query products table by id AND company_id
- Return product data or null

**Security:** Every DB query helper MUST include `.eq('company_id', companyId)` — the service role bypasses RLS, so application-level scoping is mandatory.

**Error handling:** All DB query helpers wrap in try/catch. On error, log and return null/empty array. Never throw from bridge methods.
