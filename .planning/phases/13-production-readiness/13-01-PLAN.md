  <action>
Create src/lib/env.ts with a Zod schema validating all environment variables used in the codebase:

**Required (server-side):**
- SUPABASE_SERVICE_ROLE_KEY: z.string().min(1)
- ANTHROPIC_API_KEY: z.string().startsWith('sk-')
- CRON_SECRET: z.string().min(1)

**Required (public):**
- NEXT_PUBLIC_SUPABASE_URL: z.string().url()
- NEXT_PUBLIC_SUPABASE_ANON_KEY: z.string().min(1)

**Optional (with defaults or graceful degradation):**
- NEXT_PUBLIC_SITE_URL: z.string().url().optional()
- TELEGRAM_BOT_TOKEN: z.string().optional() (legacy fallback in dispatcher)
- TELEGRAM_BOT_TOKEN_EGITIMCI through TELEGRAM_BOT_TOKEN_IADE_KALITE (12 tokens): all z.string().optional() — agents degrade gracefully if token missing
- SENTRY_DSN: z.string().url().optional() (for Plan 05)

Export two schemas:
1. `serverEnvSchema` — validates at module load time using z.object({...}).parse(process.env). Throws ZodError with formatted message listing all missing/invalid vars.
2. `publicEnvSchema` — validates NEXT_PUBLIC_* vars only (safe for client).

Export `env` as the parsed result of serverEnvSchema for type-safe access: `import { env } from '@/lib/env'` then `env.ANTHROPIC_API_KEY`.

Add a top-level `const env = serverEnvSchema.parse(process.env)` that runs at module load. Wrap in try/catch that console.error a human-readable table of missing vars and re-throws.

<task type="auto">
  <name>Task 2: Create /api/health endpoint</name>
  <files>src/app/api/health/route.ts</files>
  <action>
Create src/app/api/health/route.ts as a public GET endpoint (no auth required — already covered by middleware's /api/ exclusion).

The endpoint checks:
1. **Database connectivity**: Query `SELECT 1` via service client (supabase.from('companies').select('id').limit(1)). If fails, status = 'degraded'.
2. **Env validation**: Import and catch errors from env.ts. If env validation would fail, status = 'degraded'.
3. **Timestamp**: Include current ISO timestamp.

Response shape:
```json
{
  "status": "healthy" | "degraded",
  "timestamp": "2026-03-05T10:00:00.000Z",
  "version": "1.0.0",
  "checks": {
    "database": { "status": "ok" | "error", "latency_ms": 45 },
    "env": { "status": "ok" | "error", "missing": [] }
  }
}
```

Return HTTP 200 for healthy, HTTP 503 for degraded. Set `Cache-Control: no-store` header.

Use `performance.now()` to measure DB latency. Wrap DB check in try/catch — never throw from health endpoint.

Do NOT use `import { env } from '@/lib/env'` at the top level (that would make the route crash if env is invalid). Instead, dynamically import or use a try/catch wrapper to test env validity without crashing the route itself.
  </action>

  </verify>
