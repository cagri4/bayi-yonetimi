  <action>
Create .github/workflows/ci.yml with a 3-step CI pipeline.

```yaml
name: CI

on:
  push:
    branches: [master]
  pull_request:
    branches: [master]

jobs:
  ci:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 9

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Lint
        run: pnpm run lint

      - name: Type check
        run: npx tsc --noEmit

      - name: Build
        run: pnpm run build
        env:
          NEXT_PUBLIC_SUPABASE_URL: "https://placeholder.supabase.co"
          NEXT_PUBLIC_SUPABASE_ANON_KEY: "placeholder-key-for-build"
```

Key decisions:
- Use pnpm (matches project setup) with `--frozen-lockfile` for reproducible installs
- Node 20 (LTS, matches Vercel default)
- NEXT_PUBLIC_* env vars need placeholder values for build to succeed (Next.js inlines them at build time). Only public vars needed — server-side vars are not required at build time.
- Run order: lint -> type-check -> build (fail fast on cheapest checks)
- Trigger on push to master AND pull requests to master
- Single job (not parallel) keeps it simple — estimated ~2 min total for this project size

Do NOT add test step yet — no tests exist. Plan 06 will add test step after Vitest is configured.

Create .github/workflows/ directory if it does not exist (mkdir -p).
