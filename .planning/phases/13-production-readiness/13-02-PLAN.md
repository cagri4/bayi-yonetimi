  <files>src/lib/api-response.ts</files>
  <action>
Create src/lib/api-response.ts with helper functions for consistent API responses across all routes.

Export the following:

```typescript
import { NextResponse } from 'next/server'

type ApiSuccessResponse<T> = {
  success: true
  data: T
  timestamp: string
}

type ApiErrorResponse = {
  success: false
  error: {
    message: string
    code?: string
  }
  timestamp: string
}

export function apiSuccess<T>(data: T, status = 200): NextResponse<ApiSuccessResponse<T>>
export function apiError(message: string, status = 500, code?: string): NextResponse<ApiErrorResponse>
```

Implementation:
- `apiSuccess(data, status)` — wraps data in `{ success: true, data, timestamp: new Date().toISOString() }`, returns NextResponse.json with given status
- `apiError(message, status, code)` — wraps in `{ success: false, error: { message, code }, timestamp }`, returns NextResponse.json with given status

Also export common error shortcuts:
- `apiBadRequest(message)` — apiError(message, 400, 'BAD_REQUEST')
- `apiUnauthorized(message?)` — apiError(message || 'Unauthorized', 401, 'UNAUTHORIZED')
- `apiNotFound(message?)` — apiError(message || 'Not found', 404, 'NOT_FOUND')
- `apiRateLimited(message?)` — apiError(message || 'Too many requests', 429, 'RATE_LIMITED')

Do NOT refactor existing API routes to use these helpers in this plan — that would touch too many files. Existing routes continue working. New routes and future refactors will adopt this pattern. The helpers are standalone and ready for incremental adoption.
