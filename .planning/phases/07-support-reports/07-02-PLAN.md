1. **Create src/lib/actions/support.ts** with 'use server' at top:

   Validation schemas (use zod, following financials.ts createTransactionSchema pattern):
   - sendMessageSchema: { subject: z.string().min(1).max(200), category: z.enum(['siparis','urun','odeme','teknik','diger']), body: z.string().min(10).max(5000) }
   - productRequestSchema: { product_id: z.string().uuid().optional(), product_name: z.string().min(1).max(200), product_code: z.string().optional(), requested_quantity: z.number().int().min(1).max(9999), notes: z.string().max(1000).optional() }

   Dealer mutations:
   - `sendSupportMessage(input)`:
     - Auth check: getUser() -> if no user return { success: false, error: 'Oturum acmaniz gerekiyor' }
     - Dealer lookup: SELECT id FROM dealers WHERE user_id = user.id
     - Validate input with sendMessageSchema.safeParse
     - INSERT INTO support_messages (dealer_id, subject, category, body, status='pending')
     - revalidatePath('/support')
     - Return { success: true } or { success: false, error }

   - `submitProductRequest(input)`:
     - Same auth + dealer lookup pattern
     - Validate with productRequestSchema.safeParse
     - INSERT INTO product_requests (dealer_id, product_id, product_name, product_code, requested_quantity, notes, status='open')
     - revalidatePath('/support/product-requests')
     - Return { success: true } or { success: false, error }

   Dealer queries (no 'use server' needed, but keep in same file for co-location):
   - `getSupportMessages()`:
     - Auth check + dealer lookup
     - SELECT * FROM support_messages WHERE dealer_id = dealer.id ORDER BY created_at DESC
     - Returns SupportMessage[]

   - `getFaqWithCategories()`:
     - SELECT faq_categories with nested faq_items WHERE is_active = true ORDER BY display_order
     - Use Supabase nested select: faq_items(id, question, answer, display_order)
     - No auth required (FAQ is global, all authenticated can read)
     - Returns FaqCategoryWithItems[]

   - `getDealerProductRequests()`:
     - Auth check + dealer lookup
     - SELECT * FROM product_requests WHERE dealer_id = dealer.id ORDER BY created_at DESC
     - Returns ProductRequest[]

   Admin queries (add isAdmin check using pattern from financials.ts verifyAdmin):
   - `getAllSupportMessages()`:
     - Admin-only
     - SELECT support_messages with dealer company_name JOIN ORDER BY created_at DESC
     - Returns SupportMessageWithDealer[]

   - `getSupportMessageById(messageId: string)`:
     - Admin-only
     - SELECT single message with dealer info
     - Returns SupportMessageWithDealer | null

   Admin mutation:
   - `replyToMessage(messageId: string, replyBody: string)`:
     - Admin-only check
     - Validate replyBody: z.string().min(1).max(5000)
     - Single UPDATE on support_messages:
       SET reply_body = replyBody, replied_at = NOW(), replied_by = user.id, status = 'answered', updated_at = NOW()
       WHERE id = messageId
     - IMPORTANT: reply_body + status='answered' must be in a single UPDATE (atomic, avoids race condition noted in research pitfall 4)
     - revalidatePath('/admin/support')
     - revalidatePath(`/admin/support/${messageId}`)
     - Return { success: true } or { success: false, error }

   Admin FAQ mutations:
   - `createFaqCategory(name: string, displayOrder?: number)`: INSERT, revalidatePath('/support/faq')
   - `createFaqItem(categoryId, question, answer, displayOrder?)`: INSERT, revalidatePath('/support/faq')
   - `updateFaqItem(id, updates)`: UPDATE, revalidatePath('/support/faq')
   - `deleteFaqItem(id)`: DELETE (or set is_active=false), revalidatePath
