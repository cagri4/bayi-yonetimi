<task type="auto">
  <name>Task 1: Create migration with all Phase 6 tables</name>
  <files>supabase/migrations/007_dashboard_campaigns.sql</files>
  <action>
Create comprehensive migration file with:

1. **campaigns table**:
   - id (UUID, PK), title (TEXT NOT NULL), description (TEXT), image_url (TEXT)
   - start_date (DATE NOT NULL), end_date (DATE NOT NULL)
   - is_active (BOOLEAN DEFAULT true), created_at, updated_at
   - Index on (is_active, start_date, end_date) for date filtering

2. **campaign_products junction table**:
   - id (UUID, PK), campaign_id (FK campaigns), product_id (FK products)
   - discount_percent (DECIMAL 5,2) - optional per-product discount
   - UNIQUE(campaign_id, product_id)

3. **announcements table**:
   - id (UUID, PK), title (TEXT NOT NULL), content (TEXT NOT NULL)
   - priority (INTEGER DEFAULT 0) - higher = more prominent
   - is_active (BOOLEAN DEFAULT true)
   - published_at (TIMESTAMPTZ), expires_at (TIMESTAMPTZ)
   - created_at, updated_at
   - Index on (is_active, published_at) for active announcements query

4. **announcement_reads junction table**:
   - id (UUID, PK), announcement_id (FK announcements ON DELETE CASCADE)
   - dealer_id (FK dealers ON DELETE CASCADE)
   - read_at (TIMESTAMPTZ DEFAULT NOW())
   - UNIQUE(announcement_id, dealer_id)

5. **order_documents table**:
   - id (UUID, PK), order_id (FK orders ON DELETE CASCADE)
   - document_type (TEXT NOT NULL) - 'invoice' or 'irsaliye'
   - file_name (TEXT NOT NULL), file_path (TEXT NOT NULL), file_size (INTEGER)
   - mime_type (TEXT DEFAULT 'application/pdf')
   - uploaded_by (UUID FK users), uploaded_at (TIMESTAMPTZ DEFAULT NOW())
   - Index on order_id for quick lookup

6. **Add cargo columns to orders table**:
   - ALTER TABLE orders ADD COLUMN vehicle_plate TEXT
   - ALTER TABLE orders ADD COLUMN driver_name TEXT
   - ALTER TABLE orders ADD COLUMN driver_phone TEXT
   - ALTER TABLE orders ADD COLUMN cargo_notes TEXT

7. **dealer_spending_summary materialized view**:
   - dealer_id, company_name, month (DATE_TRUNC)
   - total_debit, total_credit, net_balance
   - Uses dealer_transactions + transaction_types from Phase 5
   - Index on (dealer_id, month DESC)
   - Comment: Refresh via pg_cron or manual trigger after transactions

8. **get_top_products_for_dealer RPC function**:
   - Parameters: p_dealer_id UUID, p_limit INTEGER
   - Returns: product_id, product_name, product_code, total_quantity, order_count
   - Aggregates from order_items joined with orders for dealer

9. **RLS Policies**:
   - campaigns: All authenticated can SELECT active campaigns
   - campaign_products: All authenticated can SELECT
   - announcements: All authenticated can SELECT active, unexpired
   - announcement_reads: Dealers can SELECT/INSERT own reads
   - order_documents: Dealers can SELECT own order docs, Admins can INSERT/UPDATE/DELETE
   - orders cargo columns: Inherit existing RLS (no change needed)

10. **Storage bucket RLS** (for order-documents bucket):
    - Dealers can SELECT files in folders matching their order IDs
    - Admins can INSERT/UPDATE/DELETE all files

Use wrapped (SELECT auth.uid()) pattern per RLS-wrapped-auth decision from Phase 4.
Add updated_at triggers for campaigns, announcements.
