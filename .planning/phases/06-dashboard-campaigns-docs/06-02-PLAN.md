   - `getSpendingSummary(dealerId)`: Query dealer_spending_summary materialized view
     - Returns: current month totals, previous month totals, YTD totals
     - Falls back to get_dealer_balance_breakdown if view not populated

   - `getRecentOrders(dealerId)`: Query orders table
     - SELECT id, order_number, status, total_amount, created_at
     - ORDER BY created_at DESC LIMIT 5
     - Include status join for display name

   - `getPendingOrdersCount(dealerId)`: Count pending orders
     - status IN ('pending', 'confirmed', 'preparing')
     - Returns single number

   - `getTopProducts(dealerId)`: Call get_top_products_for_dealer RPC
     - Returns top 5 products by order frequency
     - Includes product_name, product_code, total_quantity, order_count

3. **Helper function getDealerFromUser()**:
   - Reuse pattern from financials.ts
   - Get authenticated user, fetch dealer record
   - Return dealer.id or throw if not found

Use Supabase client from @/lib/supabase/server.
All functions are async Server Component compatible.
No 'use server' directive needed (these are queries, not mutations).
