<task type="auto">
  <name>Task 1: Create order documents Server Actions</name>
  <files>src/lib/actions/order-docs.ts</files>
  <action>
Create comprehensive order documents Server Actions:

1. **Admin mutations (with 'use server')**:

   - `uploadOrderDocument(formData: FormData)`:
     - Validate: orderId (UUID), documentType ('invoice' | 'irsaliye'), file (PDF, max 5MB)
     - Verify admin role (reuse verifyAdmin from financials.ts)
     - Generate unique path: `order-docs/{orderId}/{timestamp}-{documentType}.pdf`
     - Upload to Supabase Storage 'order-documents' bucket
     - Insert metadata into order_documents table
     - Revalidate order paths
     - Return { success: true, path } or throw error

   - `deleteOrderDocument(documentId: string)`:
     - Verify admin role
     - Get document record for file_path
     - Delete from storage
     - Delete from order_documents table
     - Revalidate paths

   - `updateCargoInfo(orderId: string, data: { vehicle_plate?, driver_name?, driver_phone?, cargo_notes? })`:
     - Verify admin role
     - Validate orderId exists
     - UPDATE orders SET vehicle_plate, driver_name, driver_phone, cargo_notes
     - Revalidate order paths
     - Return { success: true }

2. **Dealer/Shared queries**:

   - `getOrderDocuments(orderId: string)`:
     - SELECT from order_documents WHERE order_id = orderId
     - RLS automatically filters to authorized orders
     - Returns array of { id, document_type, file_name, uploaded_at }

   - `getDocumentDownloadUrl(documentId: string)`:
     - Get document record for file_path
     - Verify dealer owns the order (via RLS or explicit check)
     - Generate signed URL (60 min expiry) using supabase.storage.createSignedUrl
     - Return { url, expires_at } or throw unauthorized

   - `getCargoInfo(orderId: string)`:
     - SELECT vehicle_plate, driver_name, driver_phone, cargo_notes FROM orders
     - RLS handles authorization
     - Returns cargo info object or null if not set

3. **Validation helpers**:
   - Use Zod schema for file validation (size, type)
   - Pattern from research: uploadSchema with refinements

Follow patterns from financials.ts for admin verification and error handling.
