1. `getAnyDealerBalanceTool`: name `get_any_dealer_balance`. Description (Turkish): "Belirtilen bayinin cari bakiyesini getirir. Bayi ID ile sorgulama yapilabilir." Input: required `dealer_id` (string). Handler: verify dealer belongs to same company via `.eq('id', targetDealerId).eq('company_id', context.companyId).single()`, then call `get_dealer_balance_breakdown` RPC with the target dealer_id. This is DISTINCT from the Muhasebeci's `get_dealer_balance` which always uses `context.dealerId`. Per Pitfall 6 in research: different tool name avoids Map key collision.

2. `getDashboardSummaryTool` (GM-02 + GM-05): name `get_dashboard_summary`. Description: "Sirketin genel durumu: aktif bayi sayisi, siparis ozeti, en cok satan urunler, ve gelir bilgisi." No required input.
   Handler implementation — use DIRECT Supabase queries (NOT get_top_products or get_dealer_performance RPCs which are not company_id-scoped per Pitfall 2):
   - Calculate date range: 30 days ago to today
   - Parallel queries via Promise.all:
     a. Active dealer count: `supabase.from('dealers').select('id', { count: 'exact', head: true }).eq('company_id', context.companyId).eq('is_active', true)`
     b. Total orders (30d): `supabase.from('orders').select('id', { count: 'exact', head: true }).eq('company_id', context.companyId).gte('created_at', startDate)`
     c. Top 5 products: `supabase.from('order_items').select('product_name, quantity').eq('company_id', context.companyId)` — wait, order_items may not have company_id. Instead, JOIN through orders: query orders for the company in last 30d, collect order IDs, then query order_items. Alternatively: use `(supabase as any).from('order_items').select('product_name, quantity, order:orders!inner(company_id)').eq('order.company_id', context.companyId)` — but this may not work with Supabase query builder. SAFEST approach: query `orders` for last 30d with company_id, get order IDs, then query `order_items` IN those order IDs, aggregate in JS.
     d. Revenue: sum total_amount from the orders query above
   - Return JSON: period, active_dealers, total_orders_30d, total_revenue_30d, top_5_products (aggregated from order_items), summary text

3. `gmExportReportTool` (GM-03): name `export_report`. Description: "Sirket geneli finansal ve satis raporu olusturur." Input: optional `start_date`, optional `end_date`. Default: current month.
   Handler: company-wide report. Queries:
   - Dealer count (active)
   - Order count and total revenue for the period
   - Top products for the period (same approach as dashboard)
   Format as plain text (Telegram-safe), similar to Muhasebeci export_report but company-wide scope.
