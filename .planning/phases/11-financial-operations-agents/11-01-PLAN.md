Create `src/lib/agents/tools/muhasebeci-tools.ts` following the exact same structure as `egitimci-tools.ts`:
- Import `Tool` from `@anthropic-ai/sdk/resources/messages`, `SupabaseClient` from `@supabase/supabase-js`, `Database` from `@/types/database.types`, `AgentContext` from `../types`
- Define `HandlerFn` type alias: `(input: Record<string, unknown>, context: AgentContext) => Promise<string>`

**5 tool definitions (all exported individually for GM reuse in Plan 03):**

1. `getFinancialsTool` (MH-01): name `get_financials`. Queries `dealer_transactions` with JOIN on `transaction_types`. Input: optional `limit` (number, max 20, default 10), optional `start_date` (string), optional `end_date` (string). Description in Turkish. Scope by `context.dealerId` on the `dealer_id` column — NOT `company_id` (dealer_transactions has no company_id column per Pitfall 1 in research).

2. `getPaymentHistoryTool` (MH-02): name `get_payment_history`. Queries `dealer_transactions` filtered by transaction_type code IN ('payment', 'credit_note'). Input: optional `limit`, optional `start_date`, optional `end_date`. Scope by `context.dealerId`.

3. `getInvoicesTool` (MH-03): name `get_invoices`. Queries `dealer_invoices` scoped by `dealer_id`. Returns invoice_number, invoice_date, total_amount, file_name, created_at. NO signed URL generation — Telegram is text-only. Input: optional `limit`.

4. `getDealerBalanceTool` (MH-04): name `get_dealer_balance`. Calls `supabase.rpc('get_dealer_balance_breakdown', { p_dealer_id: context.dealerId }).single()`. Cast result to `{ total_debit: number; total_credit: number; net_balance: number }`. Add `interpretation` field: net_balance > 0 = 'Borc (bayi borcu)', < 0 = 'Alacak (bayi alacagi)', 0 = 'Sifir bakiye'. No input required (uses context).

5. `exportReportTool` (MH-05): name `export_report`. Generates plain-text formatted financial summary. Fetches balance via RPC + recent transactions. Formats as multi-line text with `=== FINANSAL RAPOR ===` header, period, totals, and transaction list. Input: optional `start_date`, optional `end_date`. Default to current month if omitted.

**Export `muhasebeciTools: Tool[]`** array containing all 5 tools.

**MH-06 — Hallucination prevention:**
Each financial tool's `description` must include a Turkish instruction reminding Claude to call the tool before stating any financial number. The `get_dealer_balance` tool description should include: "Bakiye bilgisi icin bu araci kullan. Tahmin yapma."

**Handler implementations:**
Each handler is a standalone async function receiving `(input, context, supabase)` — same pattern as `handleGetProductInfo` in egitimci-tools.ts. Error handling: return `[Hata: ...]` string on error, `[Kayit bulunamadi]` on empty results.

**Factory function:**
`export function createMuhasebeciHandlers(supabase: SupabaseClient<Database>): Map<string, HandlerFn>` — creates 5 handler entries. Handler names match tool names exactly.

Key implementation notes from research:
- `dealer_transactions` JOIN pattern: `.select('amount, description, reference_number, transaction_date, due_date, transaction_type:transaction_types(code, name, balance_effect)')`
- `dealer_invoices` select: `.select('id, invoice_number, invoice_date, total_amount, file_name, created_at')`
- Balance RPC: `(supabase as any).rpc('get_dealer_balance_breakdown', { p_dealer_id: context.dealerId }).single()`
- `export_report` text format: lines joined with '\n', amounts formatted with `.toFixed(2)`, effect mapped: 'debit' = 'Borc', else 'Odeme'
- payment_history: first query transaction_types to get IDs for 'payment' and 'credit_note', then filter dealer_transactions by those type_ids. Alternatively, use the join and filter post-query. Simplest: use the transaction_type join and filter in JS.
