  <name>Task 1: Create agent descriptions data file and session management module</name>
  <files>src/lib/sihirbaz/agents.ts, src/lib/sihirbaz/session.ts</files>
  <action>
Create two new files in src/lib/sihirbaz/:

**agents.ts:**
- Export `AGENT_DESCRIPTIONS` — an array of 12 objects, each with `role` (matching AgentRole enum values), `name` (Turkish display name), and `description` (one-sentence Turkish description of what the agent does).
- The 12 roles in order: egitimci, satis_temsilcisi, muhasebeci, depo_sorumlusu, genel_mudur_danismani, tahsilat_uzmani, dagitim_koordinatoru, saha_satis, pazarlamaci, urun_yoneticisi, satin_alma, iade_kalite.
- Use the descriptions from the research file (src/lib/sihirbaz section of 16-RESEARCH.md) as the baseline. These are business-appropriate Turkish descriptions.

**session.ts:**
- Export type `WizardSession` matching the onboarding_sessions row shape: `{ id: string, company_id: string | null, deep_link_token: string, telegram_chat_id: number | null, status: 'pending' | 'in_progress' | 'completed' | 'expired', collected_data: Record<string, unknown>, step: number }`.
- Export `loadOrCreateSession(supabase: SupabaseClient, chatId: number, text: string): Promise<WizardSession | null>`.
  - Case 1 — `/start {token}`: Extract raw token from text. The flow MUST follow this exact order:
    1. SHA-256 hash the raw token via `crypto.subtle.digest('SHA-256', new TextEncoder().encode(rawToken))` and convert to hex string.
    2. **Resume check FIRST:** Query `onboarding_sessions` WHERE `deep_link_token = rawToken`. If a session is found AND its status is NOT 'expired', return it immediately (update `telegram_chat_id = chatId` if it was null, via updateSession). This handles the re-click /start scenario where the invite is already used but the session is still active.
    3. **Only if no existing session:** Query `onboarding_invites` WHERE `token_hash = hash AND used_at IS NULL AND expires_at > NOW()`. If invalid/used/expired, return null.
    4. Create new session (company_id from invite, deep_link_token = rawToken, telegram_chat_id = chatId, status = 'in_progress', collected_data = {}, step = 0) and mark invite `used_at = NOW()`.
  - Case 2 — any other message: Query `onboarding_sessions` WHERE `telegram_chat_id = chatId AND status IN ('pending', 'in_progress')`, order by created_at DESC, limit 1, maybeSingle. Return result or null.
  - Use `(supabase as any)` cast for onboarding_invites and onboarding_sessions queries (consistent with Phase 15 pattern — tables are in DB but typed client may not expose them fully).
- Export `updateSession(supabase: SupabaseClient, sessionId: string, patch: Partial<Pick<WizardSession, 'step' | 'collected_data' | 'status' | 'company_id' | 'telegram_chat_id'>>): Promise<void>`.
  - Simple UPDATE via supabase.from('onboarding_sessions').update(patch).eq('id', sessionId).

IMPORTANT: The session lookup order in /start case is critical — steps 2-3-4 above are sequential and must NOT be reordered. Step 2 (resume check) MUST come before step 3 (invite validation) because a returning user's invite already has used_at set, so checking used_at IS NULL first would wrongly reject them.

  </verify>

<task type="auto">
  <name>Task 2: Create step handler functions with provisioning completion</name>
  <files>src/lib/sihirbaz/steps.ts</files>
  <action>
Create src/lib/sihirbaz/steps.ts:

Export `handleStep(supabase: SupabaseClient, session: WizardSession, chatId: number, text: string, botToken: string): Promise<void>`.

Implement a switch on `session.step`:

**Step 0 (Welcome):** Send greeting message in Turkish: "Hosgeldiniz! Ben Kurulum Sihirbazinim. Firmanizi dijital donusume hazirlamak icin size yardimci olacagim.\n\nOnce birkac bilgiye ihtiyacim var.\n\nFirmanizin adi nedir?" Advance to step 1 via updateSession.

**Step 1 (Firma adi):** Save text as collected_data.firma_adi. Send "Hangi sektorde faaliyet gosteriyorsunuz? (Ornek: Gida, Tekstil, Elektronik, Insaat)". Advance to step 2.

**Step 2 (Sektor):** Save text as collected_data.sektor. Send "Yaklasik kac cesit urun satiyorsunuz?". Advance to step 3.

**Step 3 (Urun sayisi):** Save text as collected_data.urun_sayisi. Send "Kac bayiniz var? (Yaklasik sayi)". Advance to step 4.

**Step 4 (Bayi sayisi):** Save text as collected_data.bayi_sayisi. Send "Dijital asistanlarinizdan en cok ne bekliyorsunuz? (Siparis takibi, stok yonetimi, muhasebe vb.)". Advance to step 5.

**Step 5 (Beklentiler):** Save text as collected_data.beklentiler. Now send the 12 agent introductions. Build intro text from AGENT_DESCRIPTIONS: each entry formatted as "${i+1}. ${name}\n${description}" joined by double newlines. Prefix with "Harika! Simdi size 12 dijital calisaninizi tanistirmak istiyorum:\n\n". Check combined message length — if > 3800 chars, split into two messages (first 6 + last 6). After intro messages, send confirmation prompt: "Tum bu dijital calisanlar 14 gun boyunca ucretsiz olarak hizmetinizde olacak.\n\nKurulumu baslatmak istiyor musunuz? (Evet / Hayir)". Advance to step 6.

**Step 6 (Confirmation):** Check if text contains "evet" (case-insensitive) or "e" or "1". If confirmed: call `runProvisioning(supabase, session, chatId, botToken)`. If rejected (hayir/h/0): send "Anlasildı. Kurulumu iptal ettiniz. Yeni bir davet linki icin yetkilinize basvurun." and update session status to 'expired'.

**Step 7+ / completed status (Post-completion):** Send "Kurulum zaten tamamlandi. Web panelinize giris yapabilirsiniz." — no state change.

**runProvisioning helper (private function in same file):**
1. Update company settings with collected data: `supabase.from('companies').update({ settings: { sektor, urun_sayisi, bayi_sayisi, beklentiler, onboarding_completed_at: new Date().toISOString() } }).eq('id', session.company_id)`.
2. Generate fresh temp password: `crypto.randomUUID().slice(0, 12)`.
3. Look up company admin user: `supabase.from('users').select('id, email').eq('company_id', session.company_id).eq('role', 'admin').single()`.
4. Reset admin password: `supabase.auth.admin.updateUserById(adminUser.id, { password: newTempPassword })`.
5. Update session: status='completed', step=7.
6. Send completion message using `adminUser.email` (available from the `select('id, email')` query in step 3): "Kurulum tamamlandi!\n\nWeb paneliniz: {NEXT_PUBLIC_APP_URL || 'https://bayi-yonetimi.vercel.app'}\n\nGiris bilgileriniz:\nE-posta: {adminUser.email}\nGecici sifre: {newTempPassword}\n\nIlk giriste sifrenizi degistirmenizi oneririz.\n\n14 gunluk ucretsiz deneme suresi baslamistir. Keyifli kullanımlar!"
7. If company_id is null (should not happen if invite validation worked), send error message and return early.

IMPORTANT: Use `(supabase as any)` for companies.update and users.select — consistent with Phase 15 pattern.
IMPORTANT: Steps 0-5 use `sendTelegramMessage` from `@/lib/telegram/send` for all bot replies.
IMPORTANT: For step 5 (agent intros), after sending the intro message, do NOT send a separate confirmation message in a different step — send both the intros and the confirmation prompt in step 5, then advance directly to step 6 (the confirmation handler). This keeps the FSM simpler: step 5 = receive beklentiler + send intros + send confirmation, step 6 = receive evet/hayir.
